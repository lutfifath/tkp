LEARNABILITY

Kemudahan Belajar

Skor: 6/10

❌ Masalah

Entry point terlalu banyak, fitur tersembunyi, hierarchy tidak jelas

💡 Solusi

3 perbaikan: Onboarding, Navigation, Visual Hierarchy

📈 Target

Time-to-first-purchase: 15 min → 5 min

1

Interactive Onboarding

❌ Masalah

User baru kesulitan memahami fitur dasar dan perbedaan badge seller

  • • Tidak ada guidance untuk fitur baru
  • • Istilah teknis membingungkan
  • • Tutorial tidak contextual

✅ Solusi

  • Progressive Onboarding: Tutorial 5 langkah dengan spotlight
  • Contextual Tooltips: Penjelasan saat encounter fitur baru
  • Interactive Glossary: Tap-to-explain istilah teknis
  • Video Tutorials: 30-second clips untuk common tasks

📊 Expected Impact:

Time-to-first-purchase: 15 min → 5 min | Onboarding completion: 30% → 70%

2

Unified Navigation

❌ Masalah

Terlalu banyak entry point untuk fitur yang sama

  • • Beli pulsa dari 5 tempat berbeda
  • • Menu "Lainnya" 30+ items tanpa struktur
  • • User bingung mau akses dari mana

✅ Solusi

  • Single Entry Point: 1 primary access per feature
  • Mega Menu Redesign: Kategorisasi jelas (Belanja, Layanan, Keuangan)
  • Search Enhancement: Suggest produk + fitur shortcuts
  • Max 2-level depth: Tidak nested terlalu dalam

📊 Expected Impact:

Navigation confusion: 45% → 15% | Feature discovery rate: +40%

3

Visual Hierarchy & Consistency

❌ Masalah

Semua elemen compete for attention

  • • User bingung mana yang penting
  • • Terlalu banyak badges per produk
  • • Inconsistent button placement

✅ Solusi

  • Visual Priority: Primary (solid), Secondary (outline), Tertiary (text)
  • Badge Hierarchy: Max 2 badges visible, rest "+2 more"
  • Color Coding: Green (action), Red (promo), Blue (info)
  • Typography Scale: Consistent heading & body sizes

📊 Expected Impact:

Task success rate: 60% → 85% | Time to understand product: 30s → 10s

EFFICIENCY

Efisiensi Penggunaan

Skor: 5/10

❌ Masalah

Checkout 9-10 langkah, pop-up mengganggu, bulk operations tidak ada

💡 Solusi

Express checkout, minimize interruptions, bulk operations

📈 Target

Checkout time: 180s → 45s | Conversion: +25%

4

Express Checkout

❌ Masalah

Checkout memerlukan 9-10 langkah dengan banyak screen transitions

  • • Review cart → pilih alamat → pilih courier
  • • Pilih payment → konfirmasi → OTP
  • • Setiap langkah = full page load

✅ Solusi

  • One-Page Checkout: Semua sections dalam 1 halaman
  • 1-Tap Express: Default settings + biometric confirm
  • Smart Defaults: AI predict courier preference
  • Progressive Disclosure: Show advanced options only when needed

📊 Expected Impact:

Checkout time: 180s → 45s | Conversion rate: +25% | Cart abandonment: 70% → 45%

5

Minimize Interruptions

❌ Masalah

Pop-ups mengganggu flow pengguna

  • • Rating pop-up setelah setiap order
  • • Banner promo menutupi konten
  • • Full-screen ads saat browsing

✅ Solusi

  • Contextual Timing: Rating request after 3rd delivery
  • Dismissible Banners: Easy close button
  • Native Ads: Inline product placement
  • Smart Frequency: Max 1 pop-up per session

📊 Expected Impact:

User frustration: -60% | Session duration: +35% | Engagement: +28%

6

Bulk Operations

❌ Masalah

Harus action satu per satu

  • • Hapus wishlist 1 by 1
  • • Add to cart repeated action
  • • Tidak bisa bulk review

✅ Solusi

  • Multi-Select: Checkbox untuk bulk actions
  • Quick Actions: Swipe gestures untuk common tasks
  • Batch Processing: Move all, delete all options
  • Smart Suggestions: "Add similar items to cart"

📊 Expected Impact:

Task completion time: -50% | Power user satisfaction: +45%

💡 Solusi

📈 Target

10-12

Error Prevention & Recovery

❌ Masalah

  • • Tidak bisa edit alamat setelah checkout
  • • Form data hilang saat back button
  • • Session timeout tanpa warning

✅ Solusi

  • Auto-Save: Form data preserved
  • Proactive Warning: Confirm before critical actions

📊 Expected Impact:

Data loss: 30% → 2% | Error recovery: >90% | Support tickets: -40%
